Blitz 使用 `app/pages/_app.ts` 中的 `App` 组件来初始化页面。你可以对齐进行
修改，来做出各种惊叹的成果：

- 在页面更改时持久化布局系统
- 浏览不同页面时保持 state
- 使用 `componentDidCatch` 自定义错误捕获逻辑
- 将其它数据注入页面
- [增加全局 CSS](./css)

默认的 `_app.tsx` 文件会像这样：

```tsx
// app/pages/_app.tsx
export default function App({ Component, pageProps }) {
  return <Component {...pageProps} />
}
```

`Component` prop 是当前活动的 `page`，所以每当你在路由之间切换时，此
`Component` 都会切换为新的 `page`。因此，任何你传递到 `Component` 的 props
都会被 `page` 接收。

`pageProps` 是一个带有初始 prop 的对象，该 prop 会通过我们的一种数据获取方
法如 [`getStaticProps`](./get-static-props) 或
[`getServerSideProps`](./get-server-side-props) 等预先加载到你的页面中。
