## Blitz 是一个社区项目 🤝 {#blitz-is-a-community-project}

Blitz [最初由 Brandon Bayer 发起](./community-history)，但从一开始它就是社
区项目。38 个伙伴为第一个 alpha 版本做出了贡献！

Blitz 不归任何公司所有，也没有筹集任何风险投资。

Brandon 大部分时间都在 Blitz 上全职工作。自 2020 年 2 月 17 日以来，他平均
每周工作 30 小时（如果你想看看为此你免费获得了多少价值，欢迎进行数学计算
😉）。然后他同时做着咨询以支持自己。

所有其余贡献者都利用空闲时间在贡献。我们现在拥有 [超过 200 个贡献
者][over 200 total contributors](https://github.com/blitz-js/blitz#contributors-)！

**因为 Blitz 是一个社区项目，整个社区都可以帮助塑造 Blitz 以及影响它的未来
发展方向。**

## 在哪里提问 {#where-to-ask-questions}

首先，搜索 [Blitz Discord](https://discord.blitzjs.com) 和
[GitHub 仓库](https://github.com/blitz-js/blitz) 来看是否有人已经询问了你
的问题。

接着
，[将你的问题发布到我们的讨论区内](https://github.com/blitz-js/blitz/discussions)。
这个相比 Discord 来说让用户更能简单的找到并用链接定位到。

然后欢迎你在 Discord 中发布指向你的讨论主题的链接，因为 Github 的讨论区目
前还没有 Discord 通知集成。

🙏 随着你在 Blitz 中逐步收获更多经验，恳请帮助回答其他人的问题！

## 如何汇报一个 Bug {#how-to-report-a-bug}

首先
，[搜索已有的 Github issues](https://github.com/blitz-js/blitz/issues?q=is%3Aissue)
来看是否有人已经汇报了你的问题。

如果没有，请
[开启一个新的 issue](https://github.com/blitz-js/blitz/issues/new/choose)！
常常会有人在一两天内回答你的新问题。

## 路线图 {#roadmap}

目前核心团队正在努力实现 1.0 版本，这主要意味着一些琐碎的工作和错误修复。
但是，我们也非常欢迎其他人继续添加主要功能！

你可以在我们的 [Github issues](https://github.com/blitz-js/blitz/issues)
中找到路线图。许多计划的功能都有 Github issues，但是并不是所有的都有必要记
录上去。

## 如何提议功能/改变 {#how-to-propose-features-changes}

我们很期待有关如何让 Blitz 变得更好的想法和建议！但是请记住我们的灵感常常
比目前的人力贡献时间多太多，很难有精力让所有想法都实现。因此如果你能同时提
交一个 PR 就更好了！

### 小改动 {#minor-changes}

对于像添加新的 CLI 标志或其它类似的小改动，只需要 [创建一个新的 Github
issue][open a new github issue](https://github.com/blitz-js/blitz/issues/new/choose)
就好。

### 其它所有：RFCs {#everything-else-rf-cs}

对于其它所有的功能，哪怕小到添加一个新的 React hook，也请将你的想法发布作
为一个 RFC（意见征求）发布到
[Github 评论区](https://github.com/blitz-js/blitz/discussions/category_choices)
中。

要发布一个 RFC，请这样做：

- 将标题名前缀定义为 `[RFC]`
- 包括如下章节
  - `### 你想要什么，为什么?`
  - `### 这和你现在正做的有什么不同？`
  - `### 可能的实现方式`
- 提交之后，也请将链接分享到 Discord 上的 `#request-for-comments` 中。

## 你的角色 {#your-role}

作为 Blitz 社区的一员，我们会很重视你的意见、想法、建议和帮助！

欢迎对任何 Github issue、拉取请求、RFC 等发表评论！整个社区都能塑造 Blitz
的发展方向！

随着你获得更多经验，如果你还能包括但不限于回答人们的问题、审阅 PR、改进文
档和提交 PR 等，我们都会非常感谢你。

## 如何在社区中升级 {#how-to-level-up-in-the-community}

你可以通过经常参与、乐于助人和友善来在我们的社区中获得尊重和影响力。

我们有两个官方的维护者等级，你可以在下面阅读，但也非常欢迎你在被官方认可之
前做维护者的工作。事实上，做维护者的工作时在社区中升级的最佳方式之一。

另一个升级的好方法是通过博客文章、视频和课程等，向人们分享 Blitz 并帮助人
们成为成功的 Blitz 开发人员。

## NPM 版本发布 {#npm-releases}

目前 Brandon 手动控制着所有的 npm 版本。如果你需要的东西已经合并到了
canary 分支，但还没有被发布在 canary 版本中，如果你向 Brandon 提到这件事，
他回很乐意发布一个更新的 canary 版本。

## 财政 {#finances}

目前，赞助和捐款的总额远低于正常的生活工资。所以目前所有的资金都流向了
Brandon。一旦 Brandon 从 Blitz 获得了全职工资，更多资金将用于支付其他也参
与到 Blitz 工作的开发者。

我们也**非常感谢使用 Blitz 的公司用薪水鼓励员工**在 Blitz 中做贡献，无论是
兼职还是全职！

## 核心团队 {#core-team}

Blitz 核心团队由对 Blitz 有持续深入投入的人组成。核心团队拥有
Github、Discord 和 npm 管理员权限。人们可以从 1 级维护者开始，成为 2 级维
护者，然后进入核心团队，从而成为核心成员。

2 级维护者是社区的中坚力量，负责审查和合并 PR。L2 是代码的守门员，确保着代
码的质量、正确性和安全性。L2 还有助于促进快速发展。

关于更多的细节，请查阅 [成为一个 L2 维护者](./maintainers)。

## L1 维护者 {#l1-maintainers}

1 级维护者主要帮助管理社区和分类问题。L1 对于健康的 Blitz 社区至关重要。L1
减轻了核心团队和 2 级维护人员的大量负担，因此后者可以专注于具有长期影响的
更高级别的事情。

关于更多细节，请查阅 [成为一个 L1 维护者](./maintainers)。
