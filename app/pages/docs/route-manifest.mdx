Blitz 为你生成 _Route 清单_。它云溪你通过 _name_ 而不是 _location_ 来引用
页面：

```ts
// 假设你在 app/pages/products/[productId].tsx 有一个页面
export default function ProductPage() { ...

// 然后你可以使用 Routes...
import { Link, Routes } from "blitz"

// ...按名称引用它...
<Link href={Routes.ProductsPage({ productId: 123 })} />

// ...而不是查找 location！
<Link href={`/products/${123}`} />
```

_路由清单_ 是一个纯粹的可选功能。，不过，它有一些有点：

- 提高表现力
- 简化将页面移动到新位置的过程

## 查询参数 {#query-parameters}

查询参数可以与路由参数一起制定

```ts
// 无须 ...
<Link href={`/products/${pid}?offerCode=capybara`} />

// ... 其实你可以这样：
<Link href={Routes.Product({ pid, offerCode: "capybara" })} />
```

## 生成清单 {#generating}

路由清单直接从你的源代码中生成到 `node_modules/.blitz` 中
。[`blitz build`](./cli-build) 和 [`blitz dev`](./cli-dev) 都会自动保持最
新。如果您需要手动生成路由清单，例如在 CI 中进行类型检查，请使用
[`blitz codegen`](./cli-codegen)。
