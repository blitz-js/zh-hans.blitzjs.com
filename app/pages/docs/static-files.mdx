---
title: 静态文件和图片
sidebar_label: 静态文件和图片
---

Blitz 可以在根目录下的 `public` 文件夹中提供例如图片一样的静态文件。在
`public` 中的文件均可以在代码中通过绝对路径 URL（`/`）来引入。

例如，如果你创建了一个 `public/my-image.png` 图片文件，下面这段代码可以正
确访问到这个图片：

```jsx
function MyImage() {
  return <img src="/my-image.png" alt="my image" />
}

export default MyImage
```

这个文件夹对于 `robots.txt`、谷歌网站验证服务（Google Site Verification）
和任何其它静态文件（包含 `.html`）来说都很有用！

> **注意**：请不要将 `public` 文件夹重命名为其它名称。该名称不能被改变，且
> 只有这个文件夹是用来服务静态文件。

> **注意**：请确保静态文件的名称和任何在 `pages/` 文件夹下的文件名称没有任
> 何同名冲突，否则会导致意外错误。
>
> 查阅更多：<http://err.sh/next.js/conflicting-public-file-page>
